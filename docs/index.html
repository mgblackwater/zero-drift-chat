features
WhatsApp Messaging
Send and receive real WhatsApp messages through a pure-Rust implementation. No Electron, no browser, no overhead.
QR Code Auth
Scan a QR code right in your terminal to pair WhatsApp. Unicode half-block rendering, no external viewer needed.
Message History
Messages persist in a local SQLite database. Scroll back through history, even after restart.
Vim Keybindings
Navigate chats with j/k, enter insert mode with i, escape back to normal mode. Feels like home.
Multi-line Input
Compose longer messages with Alt+Enter line breaks. The input area grows to fit your message.
Chat Management
Rename chats for clarity, pin favorites to the top. Your chat list, organized your way.
URL Highlighting
Links in messages are automatically detected and highlighted. Copy them from your terminal as usual.
In-App Settings
Toggle timestamps, 24-hour clock, compact mode, and more without leaving the TUI.
Smart Status Bar
Per-provider connection status with colored dots. See at a glance what's connected.
SQLite Storage
All data stored locally in ~/.zero-drift-chat/. No cloud, no sync service, your messages stay yours.

tips & limitations
  • QR scanning: Make your terminal wide enough so the QR code renders cleanly. If your phone can't read it, try increasing font size temporarily.
  • Session persistence: WhatsApp sessions are saved in ~/.zero-drift-chat/whatsapp-session.db. You only need to scan the QR code once per device pairing.
  • Chat names: WhatsApp chats appear with phone numbers by default. Use r in normal mode to rename them to something readable.
  • Pinning: Press p to pin important chats to the top of the list. Pinned chats show a * marker.
  • Pairing issues: If pairing fails, ensure your phone has an active internet connection and WhatsApp is up to date. Try quitting and restarting the app for a fresh QR code.
  • Nightly Rust: Building from source requires the Rust nightly toolchain. The included rust-toolchain.toml handles this automatically.
